🌱 EcoTrack - Carbon Footprint Awareness Platform
> A behavior-shaping system that invisibly guides people to live lower-carbon lives

CI License: MIT

EcoTrack is a modern, accessible web application that helps individuals understand, track, and reduce their personal carbon footprint through simple inputs and personalized, AI-generated insights.

Built as a single full-stack web application using Next.js (React + TypeScript), leveraging Google Gemini for personalized advice, Supabase for robust data tracking, and Clerk for secure authentication.

## 1. Chosen Vertical
Carbon Footprint Awareness Platform — a tool for everyday individuals who want to know where their emissions come from and what to actually do about them. The product is organised around three core pillars:

| Pillar | In the product |
|--------|----------------|
| **Understand** | Enter lifestyle facts → get an annual footprint broken down by category, compared to global averages and sustainable targets. |
| **Track** | Save snapshots over time to a secure database and visualize whether your footprint is trending down using interactive charts. |
| **Reduce** | Receive personalized, quantified actions generated by AI that target your biggest emission sources first. |

## 2. Approach & Logic
The decision flow (smart, context-driven assistant):

1. **User inputs** (transport, home, diet, consumption)
2. **Carbon engine** computes per-category kg CO₂e, ranking them by size.
3. **Insights generator** leverages Google Gemini to deliver tailored advice based on your largest emission sources.
4. **Snapshot history** is saved securely via Supabase for historical tracking and trend analysis.

The "logical decision making based on user context" shows up by targeting the user's own emission categories and giving actionable advice for the biggest contributors. A heavy driver is told about transport alternatives; a heavy-meat eater is told about diet adjustments.

## 3. How the Solution Works

### Architecture
*   **Frontend & API**: Next.js 16 (App Router) serving accessible UI components (Shadcn UI, Base UI), animated with Framer Motion, and charting via Recharts.
*   **Authentication**: Clerk handles user identity and session management securely.
*   **Database**: Supabase (PostgreSQL) is used for reliable, scalable data storage of footprint histories.
*   **AI Engine**: `@google/genai` is used to communicate directly with Google's Gemini models for rich, personalized emission reduction guidance.

### Project Layout
*   `frontend/` - The core Next.js application containing all React components, routes, state management (Zustand), and utility functions.
*   `.github/` - CI/CD configuration for linting, formatting, type checking, and automated tests.

### Key Technologies
*   **Framework**: Next.js, React 19, TypeScript
*   **Styling**: Tailwind CSS v4
*   **State & Data**: Zustand, Supabase SSR
*   **Testing**: Vitest, Testing Library

## 4. Running Locally

### Prerequisites
*   Node.js 20+
*   npm or pnpm
*   Supabase, Clerk, and Gemini API keys

### Setup
1. Navigate to the frontend directory:
   ```bash
   cd frontend
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Set up environment variables. Create a `.env.local` file in the `frontend` directory with the following variables:
   ```env
   NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=your_clerk_key
   CLERK_SECRET_KEY=your_clerk_secret
   NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
   NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
   GEMINI_API_KEY=your_gemini_api_key
   ```
4. Start the development server:
   ```bash
   npm run dev
   ```
   Open `http://localhost:3000` to view the application.

| Command | Purpose |
|---------|---------|

## 6. Deploying
This project is optimized for deployment on Vercel.

1. Push your code to a GitHub repository.
2. Import the project into Vercel.
3. Add the required environment variables (`NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY`, `CLERK_SECRET_KEY`, `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `GEMINI_API_KEY`).
4. Deploy! Vercel will automatically build and host the Next.js application.

## 7. Assumptions Made
*   **Awareness, not audit**: Emission factors are representative public averages for education, not certified carbon accounting.
*   **Personalization**: Gemini is used on a best-effort basis. If the AI is unreachable, the system will fall back to rule-based or generic recommendations.
*   **Privacy**: User data is isolated per Clerk user account and stored securely in Supabase.
