# 🌱 EcoTrack — Carbon Intelligence System
**License:** MIT

**Virtual PromptWars — Challenge 3.** A production-grade web app that helps individuals understand, track, and reduce their personal carbon footprint through behavior-driven inputs and personalized, AI-generated insights.

Built as a single, accessible full-stack web application using **Next.js 14 (App Router)** and **TypeScript**, leveraging **Google Gemini** for personalized advice, **Supabase** for tracking, and **Clerk** for secure identity management, deployed seamlessly to Vercel.

🔗 **Live demo**
[https://eco-track-omega-azure.vercel.app/](https://eco-track-omega-azure.vercel.app/)

## 1. Chosen vertical
**Carbon Footprint Awareness Platform** — a tool for everyday individuals who want to know where their emissions come from and what to actually do about them without tedious manual data entry. The product is organized around the three core pillars:

| Pillar | In the product |
| :--- | :--- |
| **Understand** | A conversational flow captures daily lifestyle routines (commute, diet, specific activities) and translates them into an instant CO₂ footprint. |
| **Track** | Save snapshots over time to a scalable database, allowing users to see their footprint trends on a real-time dashboard. |
| **Reduce** | Receive personalized, quantified actions generated by AI that target your specific emission patterns. |

## 2. Approach & logic

**The decision flow (smart, context-driven assistant)**
```text
User inputs (commute, diet, additional activities)
        │
        ▼
Carbon calculation engine ──► per-category kg CO₂e ──► ranked by impact
        │                                                  │
        ▼                                                  ▼
Store update & history                              Insights generator
                                              ├─ Google Gemini: tailored advice
                                              └─ Rule-based fallback: deterministic targets
        │
        ▼
Save snapshot (Supabase, keyed by Clerk user ID) → history & trend
```

The logical decision-making requested in the brief shows up in the **Insights Engine**:
1. **Gemini produces rich, context-aware advice**: A heavy driver is told about ride-hailing alternatives; a frequent flyer receives specific insights. Each recommendation carries an estimated CO₂ saving derived from the user's specific context.
2. **Graceful UI degradation**: If the AI insight fails or takes too long, the platform gracefully degrades to show calculated baseline metrics to ensure the user always receives quantified guidance.

**Emission model**
Footprint figures use scientifically backed emission factors (documented inline in `src/constants/emissionFactors.ts`). Every calculation is rigorously abstracted and avoids magic numbers, ensuring quantities are accurate and normalized to kg CO₂e.

## 3. How the solution works

### Architecture

```text
Browser (Next.js Client Components)      Next.js Server (Server Actions / API)
  • accessible UI & glassmorphism    ──HTTP──► Next.js API Routes
  • Clerk secure auth context          ├─ POST /api/ai/insights (Gemini orchestration)
  • Zustand (state management)         ├─ Supabase Client (Data persistence)
                                       └─ Zod Validation (Type-safe payloads)
                                              │
                                              ├─► Google Gemini (via API)
                                              └─► Supabase (PostgreSQL)
```
One framework (Next.js) serves both the robust API and the highly interactive SPA, deployed globally on Vercel. 

### Project layout
- `src/app/` — Next.js App Router (pages, layouts, API routes).
- `src/components/` — React UI components (Tailwind CSS, Framer Motion, Radix UI).
- `src/constants/` — Static data (emission factors, strict constants).
- `src/hooks/` — Extracted business logic (`useAIFlow`, `useCarbonCalculation`).
- `src/lib/` — API clients (`api.ts` with Zod validation) and utility functions.
- `src/store/` — Zustand store for predictable state management.

### Key endpoints
| Method & path | Purpose |
| :--- | :--- |
| `POST /api/ai/insights` | Personalized reduction advice orchestrated via Gemini. |

## 4. Running locally

**Prerequisites**: Node.js 18.17+ and npm.

```bash
# Clone the repository
git clone https://github.com/Priyadharshan2003/EcoTrack.git
cd EcoTrack

# Install dependencies
npm install

# Set up environment variables
cp .env.example .env.local
# Add your NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY, CLERK_SECRET_KEY, 
# NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY, and GEMINI_API_KEY.

# Run the development server
npm run dev
# Open http://localhost:3000
```

## 7. Assumptions made
- **Awareness, not certified audit**: Emission factors are representative public averages for education, not certified carbon accounting. Grids and lifestyles vary, so figures are estimates.
- **Micro-interventions over massive overhauls**: The UI encourages small, daily behavior adjustments rather than overwhelming the user with massive lifestyle changes.
- **Gemini is best-effort**: The application handles AI latency gracefully by abstracting the flow state (`useAIFlow`) from the asynchronous data fetching.
